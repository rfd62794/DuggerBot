# ADR-003: Tower is the Authoritative Twin

**Status:** Accepted
**Date:** 2026-06-15
**Author:** Robert Floyd Dugger

## Context

Two instances (Tower, Nitro 5) share one codebase. Conflicts must have a defined resolution.

## Decision

Tower (production, INSTANCE_ROLE=production) is always authoritative for production tasks. Nitro 5 (development, INSTANCE_ROLE=development) assists and tests but never overrides Tower. Nitro 5's database reads are always from Tower's SQLite. Nitro 5 never writes to Tower's production database.

## Consequences

- Any directive that gives Nitro 5 write access to Tower's production data violates this ADR.
- Tower's ledger is the source of truth for cost tracking.
- Nitro 5 may read Tower's state but never mutate it.
