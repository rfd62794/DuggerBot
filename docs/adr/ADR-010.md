# ADR-010: Six-Level Agent Hierarchy with Cline as Local Coding Sub-Agent

## Status
Accepted

## Context

ADR-008 established a five-level hierarchy. Cline + Ollama (confirmed working
on Nitro 5 with Qwen models in June 2026) introduces a viable local coding
sub-agent tier that belongs between Claude and Devin. The case for insertion:

- Cline runs headless via CLI (Cline CLI 2.0, 2026)
- Local Qwen models (qwen2.5-coder:7b, qwen3:4b) run on Nitro 5's RTX 3050 Ti
- Zero marginal cost per coding task vs Devin's cloud cost
- Appropriate for bounded, single-file, well-specified work

This supersedes ADR-008's five-level hierarchy for the agent chain only.
All other ADR-008 decisions (twin deference, scheduling, Nitro 5 primacy) remain.

## Decision

### The Six-Level Hierarchy

Authority flows top to bottom. Each level receives direction from above
and may delegate downward. No level initiates contact with a level above it.

```
1. Robert          — Human. Absolute authority.
2. Claude          — Director/Architect. Writes directives. Verifies results.
3. Cline + Ollama  — Local coding sub-agent. Bounded tasks. Free. Nitro 5.
4. Devin           — Cloud coding agent. Complex/multi-file. Escalation only.
5. Local TOBOR     — Nitro 5 MCP server. Runtime tool layer.
6. Remote TOBOR    — Tower MCP server. Background infrastructure.
```

Levels 1–4 are the development chain. Levels 5–6 are the runtime chain.

### Routing Rule (Phase 3.8 scope)

**Dispatch is manual in Phase 3.8.** Human (Robert) or Claude decides which
level handles a given task. The `dispatch_to_cline` tool (Phase 3.8) is dumb —
it executes what it is given. It does not route autonomously.

Automated routing (TOBOR evaluates task complexity and dispatches) is
Phase 4+ scope. Do not anticipate it in Phase 3.8 code.

**Advisory heuristic for manual dispatch decisions:**

| Task type | Route to |
|---|---|
| Single-file, bounded, well-specified | Cline + Ollama |
| Multi-file, architectural, reasoning-heavy | Devin |
| Test addition for existing component | Cline + Ollama |
| New phase implementation | Devin |
| Bug fix in isolated module | Cline + Ollama |
| ADR or SDD writing | Claude directly |

These are advisory. Claude and Robert apply judgment. The heuristic is a
starting point, not a rule the system enforces.

### Cline Model Preference (Nitro 5)

Primary: `qwen2.5-coder:7b` (best coding quality for 4GB VRAM, tight fit)
Fallback: `qwen3:4b` (comfortable fit, hybrid thinking, adequate quality)

Switch to fallback if primary causes VRAM pressure or crashes.

### .clinerules as Governance

`.clinerules` at the repo root is the authoritative governance document for
any Cline session on this repo. It must be present before Cline is used.
Cline sessions without `.clinerules` are not permitted on this repo.

## Consequences

**This ADR locks:**
- Six-level hierarchy replacing ADR-008's five-level hierarchy
- Cline + Ollama as Level 3 — permanent position
- Dispatch is manual through Phase 3.8
- `.clinerules` required for all Cline sessions on this repo
- Devin handles escalations, not defaults

**This ADR does NOT lock:**
- Which specific Qwen model Cline uses — operator decision per session
- Automated routing logic — Phase 4+ decision
- Whether DUGGERWORKSHOP becomes a seventh level — deferred

**Failure modes accepted:**
- Cline produces work below the 200/0/0 floor. Resolution: human reviews
  Cline output before merging. Cline output is never auto-committed.
- Qwen 7b overflows 4GB VRAM. Resolution: switch to qwen3:4b per model
  preference rule above.
